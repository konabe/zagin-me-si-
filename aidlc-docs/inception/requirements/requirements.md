# Requirements — zagin-me-si-（銀座 飯 探し）

**作成日**: 2026-05-27
**深さレベル**: Standard（標準）
**プロジェクトタイプ**: Greenfield

---

## 1. プロジェクト概要

### 1.1 名称
zagin-me-si-（銀座 飯 探し）

### 1.2 目的
職場が銀座にある人が、ランチ場所を素早く・気軽に探せるWebアプリ。

### 1.3 ターゲットユーザー
- 銀座エリアで働く社会人
- 平日のランチ選びに迷っている人
- 認証不要でサクッと使いたい人

---

## 2. 機能要件（Functional Requirements）

### FR-1: ランチ店の一覧表示
- アプリ起動時、銀座エリアのランチ店一覧を表示する
- 1店舗あたり以下を表示:
  - 店名
  - ジャンル
  - 予算帯（昼）
  - 簡単な紹介文
  - （あれば）銀座の何丁目か等の位置情報

### FR-2: ジャンルによる絞り込み
- ユーザーはジャンル（例: 和食、洋食、中華、イタリアン、カフェなど）で店舗を絞り込める
- 複数ジャンルの選択可（任意）

### FR-3: 予算による絞り込み
- ユーザーは予算帯（例: 〜1,000円 / 1,000〜2,000円 / 2,000〜3,000円 / 3,000円〜）で店舗を絞り込める

### FR-4: 絞り込みの組み合わせ
- ジャンルと予算を組み合わせて絞り込み可能
- 絞り込み結果は即座にUIに反映される

### FR-5: 結果ゼロ件のハンドリング
- 条件に該当する店舗が無い場合、その旨をユーザーに分かりやすく伝える

---

## 3. 非機能要件（Non-Functional Requirements）

### NFR-1: パフォーマンス
- 初回ページ表示: 2秒以内（ローカル環境）
- 絞り込み操作のレスポンス: 即座（クライアントサイドフィルタで十分）

### NFR-2: データ
- 店舗データは自前のJSON/シードデータとして同梱
- 初期サンプル件数: 10〜20件程度（MVPとして必要十分）
- データソース切り替えを将来想定しなくてもよい（MVPスコープ外）

### NFR-3: 認証・セキュリティ
- ユーザー認証は **なし**
- 個人情報の取り扱いなし
- セキュリティベースライン拡張は **適用しない**（PoC / プロトタイプ扱い）

### NFR-4: 対応環境
- モダンブラウザ（Chrome / Edge / Safari の最新版）
- スマホでの閲覧も想定（レスポンシブ）

### NFR-5: テスト
- ユニットテスト・統合テストは実施
- プロパティベースドテスト（PBT）拡張は **適用しない**

### NFR-6: 技術スタック
- TypeScript / Node.js 系
- 具体のフレームワーク選定は NFR Requirements ステージで実施

---

## 4. スコープ

### 4.1 含むもの（In Scope）
- 銀座のランチ店一覧表示
- ジャンル絞り込み
- 予算絞り込み
- 上記のレスポンシブUI
- サンプル店舗データ（JSON）

### 4.2 含まないもの（Out of Scope）
- ユーザー登録・ログイン
- お気に入り / 訪問履歴
- 地図表示
- 「今日のランチ」ランダム提案
- レビュー機能
- 多人数共同利用機能
- 外部API連携（Google Maps、食べログ等）
- 決済・予約

---

## 5. 受け入れ基準（Acceptance Criteria）

- [ ] AC-1: アプリにアクセスすると、銀座のランチ店一覧が表示される
- [ ] AC-2: ジャンルを選択すると、該当店舗のみに絞り込まれる
- [ ] AC-3: 予算帯を選択すると、該当店舗のみに絞り込まれる
- [ ] AC-4: ジャンルと予算を同時に指定して絞り込める
- [ ] AC-5: 該当店舗がない場合、その旨が表示される
- [ ] AC-6: スマホでも崩れず使える

---

## 6. 前提と制約

- 銀座エリアの実在店舗データは正確性を保証しない（サンプル扱い）
- API キーや外部サービスは使わない
- 1人のユーザー（自分）向けでもよく、認証や同時アクセス考慮は不要

---

## 7. Extension 設定

| Extension | Enabled | 理由 |
|-----------|---------|------|
| Security Baseline | No | PoC / プロトタイプとして実装。認証なし、個人情報なしのため |
| Property-Based Testing | No | シンプルなCRUD/フィルタ処理のため、通常テストで十分 |
